# 外贸独立站全栈项目 数据库设计文档

## 一、数据库概述

- **数据库类型**：SQLite 3
- **字符编码**：UTF-8（SQLite 默认支持 Unicode）
- **表数量**：7 张核心表
- **设计原则**：
  - 每张表包含创建时间、更新时间、软删除字段
  - 合理索引优化查询性能
  - 多语言字段使用 JSON 格式存储
  - 遵循第三范式设计

## 三、表结构详细设计

### 3.1 管理员表 (admin)

| 字段名 | 类型 | 长度 | 必填 | 默认值 | 说明 |
|--------|------|------|------|--------|------|
| id | INTEGER | - | 是 | - | 主键，自增 |
| username | VARCHAR | 50 | 是 | - | 用户名，唯一 |
| password_hash | VARCHAR | 255 | 是 | - | 密码哈希（bcrypt） |
| nickname | VARCHAR | 50 | 否 | NULL | 昵称 |
| avatar | VARCHAR | 255 | 否 | NULL | 头像URL |
| status | TINYINT | 1 | 是 | 1 | 状态：0禁用，1启用 |
| created_at | DATETIME | - | 是 | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | - | 是 | CURRENT_TIMESTAMP | 更新时间 |
| deleted_at | DATETIME | - | 否 | NULL | 删除时间（软删除） |

**索引：**
- PRIMARY KEY (`id`)
- UNIQUE INDEX `idx_username` (`username`)
- INDEX `idx_status` (`status`)
- INDEX `idx_deleted_at` (`deleted_at`)

## 四、多语言字段设计说明

所有多语言字段统一使用 JSON 格式存储，结构如下：

```json
{
  "zh-CN": "中文内容",
  "en-US": "English content"
}
```

**使用的表和字段：**
- `category.name` - 分类名称
- `product.name` - 产品名称
- `product.description` - 产品描述
- `product.packaging` - 包装信息
- `news.title` - 新闻标题
- `news.content` - 新闻内容
- `banner.title` - 轮播标题
- `site_config.site_name` - 网站名称
- `site_config.site_title` - 网站标题
- `site_config.keywords` - SEO关键词
- `site_config.description` - SEO描述
- `site_config.address` - 公司地址
- `site_config.about_us` - 关于我们
